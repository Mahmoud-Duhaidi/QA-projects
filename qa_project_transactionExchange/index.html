Setup

The project is divided into two sections: client/ and server/.

Client Folder

The client/ folder contains the front-end part of the project:

  • To run the front-end, use the "Live-server extension on VSCODE". This will initiate a local server to view the front-end part by default on 127.0.0.1:5500/client/index.html.
  • (Optional) Use a server to host the front-end part.
  • This section is implemented using HTML5, CSS3, Vanilla JavaScript (ES6+), and Bootstrap 5 for CSS customization.

Server Folder

The server/ folder contains the back-end part and services, including the API:

  • The backend is implemented in Java, mainly for the use of testing tools such as JUnit and Mockito.
  • The connection between the client-server architecture is using API endpoints, which are implemented in Java using the Spring Boot framework.
  • To run the server, you can use the Spring Boot development server located on 127.0.0.1:8080. This can be done by initializing a Spring Boot project using IntelliJ IDEA, the Spring Initializr, or by opening the existing project and then running the QaprojApplication.class main class to run the server.
  • The API endpoints are located in "QaprojApplicationController" controller class:

    1. @POST METHOD convert(@RequestBody ConversionRequest) : ResponseEntity : takes input from front-end and execute convert:
      • fetches data from a real database such as : rate between the two currencies chosen
      • calculates the rate * amount and returns the converted data From-To


    2. @GET METHOD getRate(@RequestParam String , @RequestParam String ) : Mono> wheic takes the To-From currencies codes and returns the Rate only between them

in summary, running the front-end (live-server) and the back-end server using spring boot gives you a simple full-stack Money Exchange project to be used

TESTING:

we used two testing methods to test the web interface and the API methods using:

  1. Junit
  2. Mockito
  3. selenium IDE & selenium driver.

step 1-2 :

to use Junit testing and Test the pre-written tests (under server/qaproj/src/test/java/junitMockito/ ): if you already have installed the dependencies & plugins in the pom.xml file then: (note backend & front-end server should be running)

  1. run the ConversionServiceTest.class to run the tests in the class
  2. a report file was pre-generated for this cases :report file can be alocated in: ../testing Reports/Junit&Mockito/site/surefire-report.html

step 3 :

to use Selenium framework testing and test the pre-written tests ( under: server/qaproj/src/test/java/selenium/):

  1. you can locate a file called "qaproject_MoneyExchange.side" this is a Selenium IDE file of a pre-existing selenium web tests
  2. or run the ExtentReportsTests.class to execute selenium tests and get the generated tests report
  3. a selenium extent pre-generated report for this cases can be located under: ../testing Reports/selenium/testReport.html
